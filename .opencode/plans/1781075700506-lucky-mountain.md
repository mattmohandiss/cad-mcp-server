# Final MCP Tool Surface: 4 Deterministic Fact Tools

## Decision
- **Remove `query_step_features`** ‚Äî it was analysis (confidence scores, candidates) masquerading as facts. Full stop, already done.
- **Add `has_inner_wires`** to `query_step_faces` ‚Äî the ONE missing fact that makes the feature detector fully redundant.
- **4-tool surface**: `inspect_step_file`, `query_step_faces`, `query_step_edges`, `compare_step_files`

## Why Just `has_inner_wires`?

| Use Case | Round Trips | How |
|---|---|---|
| Through vs blind hole | 2 | (1) query cylinders + adjacent_faces, (2) query planar neighbor IDs with `has_inner_wires` |
| Fillet detection | 1-2 | Query circular edges + adjacent_faces, filter out cylinder-pair adjacencies |
| Pocket detection | 1 | Query planar + all-concave adjacencies + `has_inner_wires=false` |
| Hole axis | Inferred | LLM derives from adjacent planar face normals (in `adjacent_faces`) |

**Not needed** (can be composed by LLM from existing data):
- `shared_curve_types` ‚Äî saves 1 round trip for fillet confirmation, not required
- Cylinder axis ‚Äî derivable from adjacent planar normals
- `vexity` on edge `adjacent_faces` ‚Äî not needed for common workflows

**Design choice**: `has_inner_wires` is a top-level face include, NOT embedded in `adjacent_faces`. Reasoning:
- Multi-turn transparency: engineer sees "face:3 has inner wires" as an explicit intermediate step
- Clean data ownership: inner wires are a property of the face, not of the adjacency
- 2 round trips is acceptable for an interactive chat workflow
- Minimal context: only returned when explicitly requested
