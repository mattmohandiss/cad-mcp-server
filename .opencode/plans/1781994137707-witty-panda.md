# Stripped read-only OCCT wasm kernel for STEP analysis

Build a minimal WebAssembly CAD kernel (read-only STEP analysis) by stripping OCCT from ~40 toolkits to ~13. Remove unused facade methods (modeling, visualization, XCAF). The MCP server only reads STEP files and returns raw geometric facts — all inference and analysis is done by the LLM, never by the tools.

## Architecture

Two non-negotiable rules:
1. **MCP tools deal only in geometric facts.** Surface type, area, length, normal, curvature, adjacency (which faces share an edge), PMI values — raw data only. No feature recognition, no manufacturability scoring, no classification.
2. **All inference and analysis falls on the LLM.** The LLM composes primitive tool calls to answer engineer questions. "Find through-holes" = find cylindrical faces → check wire count on each. "Wall thickness" = find opposite face pairs → measure distance. Each query is its own MCP call.

## Research: what engineers ask

We surveyed 14 categories of mechanical engineer questions (~100+ use-cases). The LLM can answer ALL of them by composing the existing 6 MCP tools — each returns raw geometric facts:

| Engineer asks | LLM strategy using existing tools |
|---|---|
| "Is this part valid?" | `inspect_step_file` → check `health.is_valid`, face/edge counts, warnings |
| "What are the dimensions?" | `inspect_step_file` → `size.bounding_box`, `size.dimensions` |
| "Find all holes" | `find_step_faces` `surface_types:['cylinder']` → check axis, radius, `has_inner_wires` |
| "Find thin walls" | `find_step_faces` → get face pairs via `adjacent_faces` → `get_step_entities` with `closest_face_distance` |
| "Are there sharp corners?" | `find_step_edges` with small `radius.max` → `get_step_entities` for adjacent faces → check dihedral angle |
| "Draft angle on this face?" | `get_step_entities` with field `normal` → LLM computes `acos(dot(normal, pull_dir))` |
| "What changed from v1 to v2?" | `compare_step_files` → metric deltas (volume, area, body count). LLM can drill into specific bodies |
| "What tolerances exist?" | `query_step_pmi` → filter by `pmi_types`, `tolerance_subtypes`, `value_min`/`value_max` |
| "Is this face parallel to that one?" | `get_step_entities` on both faces → compare normals |
| "Are there undercuts?" | `find_step_faces` with `normal` filter → check angle to pull direction |

All 13 kept OCCT toolkits are sufficient for these queries. The only notable gap is assembly interference (`BRepAlgoAPI_Common` in TKBO — dropped), but distance-based checks via `BRepExtrema_DistShapeShape` (TKTopAlgo, kept) serve for clearance analysis.

## Plan overview

1. Create a custom builder Dockerimage with only 13 OCCT toolkits
2. Remove unused facade methods (primitives, booleans, fillets, sweeps, XCAF, HLR)
3. Expand the link exclusion list in xtask/build.rs
4. Remove unused TypeScript wrapper methods
5. Rebuild wasm
6. Update MCP server to match (fixtures, tests)

## Step 1: Builder Dockerimage with stripped OCCT

**File: `kernel/Dockerfile.builder` (create fresh)**

```dockerfile
FROM emscripten/emsdk:5.0.3

RUN apt-get update && apt-get install -y --no-install-recommends \
    cmake ninja-build \
    && rm -rf /var/lib/apt/lists/*

COPY rust-toolchain.toml /tmp/rust-toolchain.toml
RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | \
    sh -s -- -y --default-toolchain "$(grep channel /tmp/rust-toolchain.toml | cut -d'"' -f2)" \
    && rm /tmp/rust-toolchain.toml
ENV PATH="/root/.cargo/bin:${PATH}"

WORKDIR /workspace

# OCCT source + build — only 13 toolkits needed
COPY occt/ occt/
RUN mkdir -p occt/build && cd occt/build \
    && emcmake cmake .. \
        -G Ninja \
        -DCMAKE_BUILD_TYPE=Release \
        -DBUILD_MODULE_FoundationClasses=FALSE \
        -DBUILD_MODULE_ModelingData=FALSE \
        -DBUILD_MODULE_ModelingAlgorithms=FALSE \
        -DBUILD_MODULE_DataExchange=FALSE \
        -DBUILD_MODULE_ApplicationFramework=FALSE \
        -DBUILD_MODULE_Visualization=FALSE \
        -DBUILD_MODULE_Draw=FALSE \
        -DBUILD_ADDITIONAL_TOOLKITS="TKernel;TKMath;TKG2d;TKG3d;TKGeomBase;TKBRep;TKTopAlgo;TKGeomAlgo;TKMesh;TKShHealing;TKXSBase;TKDESTEP;TKDESTL" \
        -DBUILD_LIBRARY_TYPE=Static \
        -DUSE_FREETYPE=OFF \
        -DUSE_RAPIDJSON=OFF \
        "-DCMAKE_C_FLAGS=-fwasm-exceptions -O3 -msimd128 -DIGNORE_NO_ATOMICS=1 -DOCCT_NO_PLUGINS" \
        "-DCMAKE_CXX_FLAGS=-fwasm-exceptions -O3 -msimd128 -DIGNORE_NO_ATOMICS=1 -DOCCT_NO_PLUGINS" \
        -Wno-dev \
    && cmake --build . --parallel \
    && echo "OCCT: $(ls -1 lin32/clang/lib/*.a 2>/dev/null | wc -l) static libs"
```

**Build it** (takes ~30-45 min):
```bash
cd kernel && podman build -t localhost/occt-wasm-builder -f Dockerfile.builder .
```

## Step 4: Update generated C++ codegen config

**File: `kernel/xtask/src/codegen/config.rs`**

Remove all `MethodSpec` entries for methods being dropped. A quick way to identify them: any entry whose `includes` reference a dropped toolkit's headers. Run `grep -n` for these headers to find the entries:

Dropped toolkit headers (search for these in config.rs):
- `BRepPrimAPI_` → TKPrim (primitives)
- `BRepAlgoAPI_`, `BOPAlgo_` → TKBO (booleans)
- `BRepFilletAPI_` → TKFillet (fillets/chamfers)
- `BRepOffsetAPI_`, `BRepOffset_MakeOffset` → TKOffset (sweeps, shells)
- `BRepFeat_` → TKFeat (features — already excluded)
- `HLRAlgo_`, `HLRBRep_` → TKHLR (projection)
- `XCAFDoc_`, `TDF_`, `TDocStd_`, `TDataStd_`, `TPrsStd_`, `RWGltf_`, `STEPCAFControl_` → XCAF
- `GC_MakeArcOfCircle` → already in TKGeomAlgo (kept)
- `BinTools` → TKBRep (kept)

After removing entries, run `cargo xtask codegen` to regenerate `kernel.cpp` and `bindings.cpp`. This requires a Rust toolchain with `wasm32-unknown-unknown` target.

If you don't have cargo, you can manually edit `kernel/facade/generated/kernel.cpp` and `kernel/facade/generated/bindings.cpp` to remove the corresponding implementation and binding lines. Search for the method names.

## Expected results

- OCCT static libs: from ~40 toolkits → ~13 toolkits
- Builder image build time: from ~60-90 min → ~30-45 min
- Wasm binary: from ~22 MB → ~8-12 MB
- wasm-opt time: from ~15-20 min → ~3-5 min (super-linear on smaller binary)
- MCP tool surface: unchanged (inspect, find_faces, find_edges, get_entities, PMI, compare) — all 24 existing tests must pass
- No reduction in analysis capability — LLM composes the same tools to answer engineer questions
